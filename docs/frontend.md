## 模块功能

### 项目简介
"今天吃什么"（WhatToEat）是一款基于微信小程序的"干饭防纠结"应用，主要面向在校大学生及周边职场人群。针对用户日常就餐时"不知道吃什么"的痛点，应用提供了多种趣味防纠结功能。

### 核心功能模块
1. **随机抽取功能** - 通过大转盘或摇一摇方式随机选择餐厅
2. **卡片滑动选择** - 类似探探的左右滑动交互选择心仪餐厅
3. **UGC内容管理** - 用户可自定义录入校园周边美食数据
4. **标签与排雷系统** - 用户可为餐厅打标签及拉黑排雷
5. **个性化推荐** - 基于用户历史选择偏好推荐餐厅

### 页面功能
1. **首页** - 应用主界面，展示核心功能入口
2. **随机抽取页面** - 大转盘和摇一摇随机选择餐厅
3. **卡片滑动页面** - 左右滑动选择喜欢的餐厅
4. **UGC内容管理页面** - 用户添加和管理餐厅信息（待开发）
5. **个人中心页面** - 用户收藏和偏好设置（待开发）

## 目录结构

```
frontend/
├── app.js              # 小程序入口文件，注册小程序
├── app.json            # 小程序全局配置，包括页面路径、窗口样式等
├── app.wxss            # 全局样式文件
├── project.config.json # 项目配置文件，微信开发者工具相关配置
├── project.private.config.json # 项目私有配置
├── sitemap.json        # 搜索索引配置
├── .eslintrc.js        # ESLint代码规范配置
├── components/         # 自定义组件目录
│   └── navigation-bar/ # 导航栏组件
│       ├── navigation-bar.js
│       ├── navigation-bar.json
│       ├── navigation-bar.wxml
│       └── navigation-bar.wxss
└── pages/              # 页面目录
    ├── index/          # 首页
    │   ├── index.js
    │   ├── index.json
    │   ├── index.wxml
    │   └── index.wxss
    ├── random/         # 随机抽取页面
    │   ├── random.js
    │   ├── random.json
    │   ├── random.wxml
    │   └── random.wxss
    └── swipe/          # 卡片滑动页面
        ├── swipe.js
        ├── swipe.json
        ├── swipe.wxml
        └── swipe.wxss
```

### 目录说明
1. **根目录文件**
   - `app.js`: 小程序生命周期管理、全局数据、全局函数
   - `app.json`: 配置页面路径、窗口样式、tabBar等
   - `app.wxss`: 全局样式，可被所有页面引用

2. **components目录**
   - 存放可复用的自定义组件
   - 当前已有导航栏组件，后续可添加餐厅卡片、大转盘等组件

3. **pages目录**
   - 每个页面一个独立目录，包含页面的四个基本文件
   - 遵循微信小程序"页面即目录"的设计原则

## 运行方式

### 环境要求
1. **操作系统**: Windows 10/11, macOS, Linux
2. **微信开发者工具**: 最新稳定版
3. **Node.js**: 14.0.0 或更高版本（用于ESLint等工具）

### 开发环境搭建
1. **安装微信开发者工具**
   - 从微信官方下载并安装微信开发者工具
   - 使用微信扫码登录

2. **导入项目**
   - 打开微信开发者工具
   - 选择"导入项目"
   - 选择项目根目录（包含`project.config.json`的目录）
   - 填写AppID（可使用测试号）

3. **代码规范检查**
   ```bash
   # 安装ESLint（如果尚未安装）
   npm install -g eslint
   
   # 在项目根目录运行
   eslint frontend/
   ```

### 开发流程
1. **本地开发**
   - 使用微信开发者工具进行代码编辑和实时预览
   - 在模拟器中测试功能
   - 使用真机调试功能在手机上测试

2. **代码编写规范**
   - 遵循ESLint配置的代码规范
   - 组件化开发，提高代码复用性
   - 合理使用WXSS的rpx单位实现响应式布局

3. **调试技巧**
   - 使用`console.log()`输出调试信息
   - 利用微信开发者工具的调试工具
   - 使用`wx.setStorageSync()`和`wx.getStorageSync()`调试本地存储

### 构建与发布
1. **开发版本**
   - 在微信开发者工具中点击"预览"，生成开发版二维码
   - 扫描二维码在手机上测试

2. **体验版本**
   - 点击"上传"，填写版本号和项目备注
   - 在微信小程序后台设置为体验版
   - 分享体验版二维码给测试人员

3. **正式发布**
   - 确保所有功能测试通过
   - 提交代码审核
   - 审核通过后发布正式版

### 常见问题解决
1. **页面不显示**
   - 检查`app.json`中的页面路径是否正确
   - 检查页面文件是否存在且命名正确

2. **样式不生效**
   - 检查WXSS文件是否被正确引用
   - 检查样式选择器是否正确
   - 检查样式优先级

3. **API调用失败**
   - 检查网络连接
   - 检查域名是否在微信小程序后台配置
   - 检查API接口地址是否正确

### 性能优化建议
1. **图片优化**
   - 使用合适的图片格式（推荐WebP）
   - 压缩图片大小
   - 使用懒加载技术

2. **代码优化**
   - 合理使用`setData()`，避免频繁调用
   - 使用分包加载减少首次加载时间
   - 清理无用代码和资源

3. **网络优化**
   - 合并API请求
   - 使用缓存减少重复请求
   - 优化数据格式，减少传输量
