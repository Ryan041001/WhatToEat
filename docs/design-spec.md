# 干饭防纠结 - 设计说明文档

## 设计说明

本应用是一款面向在校大学生和周边职场人群的就餐选择辅助工具，采用微信小程序风格设计，通过趣味化交互帮助用户解决"不知道吃什么"的选择困难问题。

---

## 配色方案

- **主色（橘子橙）**: #F97316 - 用于核心按钮、主要操作、品牌色
- **辅助色（蓝莓紫）**: #4361EE - 用于次要操作、功能模块区分
- **强调色（牛油果绿）**: #84A98C - 用于成功提示、积极反馈
- **主背景（米饭白）**: #F8F7F4 - 应用主背景色
- **正文（芝麻灰）**: #5C5C5C - 主要文本内容
- **分割线（淡云灰）**: #E8E8E8 - 卡片边框、分隔线

**设计原则**：
- 不使用渐变色，保持扁平化设计风格
- 色彩搭配呼应"美食"主题（橘子、蓝莓、牛油果、米饭、芝麻）
- 高对比度保证移动端可读性

---

## 字体

- **标题**: System Font (iOS) / Roboto (Android) - 加粗 700
- **正文**: System Font (iOS) / Roboto (Android) - 常规 400
- **辅助文本**: 11-12px 小字号，用于次要信息

## 核心页面

### 1. 首页（Home）- 快速决策中心
**功能说明**：
- 展示应用核心功能入口（大转盘、摇一摇、卡片滑选）
- 快速随机选择按钮
- 数据统计展示（总餐厅、可选择、已拉黑）
- 热门推荐列表（按评分排序 Top 3）
- 快速添加餐厅入口

**设计特点**：
- 头部橙色区块采用装饰性圆形元素增强视觉趣味
- 三大功能按钮使用不同颜色区分：蓝莓紫（大转盘）、橘子橙（摇一摇）、牛油果绿（卡片滑选）
- 白色卡片与橙色背景形成层叠效果（负边距 -mt-8，圆角 rounded-3xl）

### 2. 大转盘（Spin）- 视觉化随机抽取
**功能说明**：
- 可视化转盘界面，展示所有可选餐厅
- 点击"开始抽取"按钮触发旋转动画
- 抽中后显示结果卡片（餐厅名称、图片、详细信息）
- 支持"再转一次"和"就这家了"操作

**设计特点**：
- 使用 Canvas 绘制动态转盘
- 平滑的缓动动画（easeOutQuint）
- 结果卡片从底部滑入（motion 动画）

### 3. 卡片滑选（Swipe）- Tinder式交互
**功能说明**：
- 类似探探的左右滑动卡片交互
- 右滑表示"想吃"，左滑表示"不想吃"
- 滑完所有卡片后随机选择一个"想吃"的餐厅
- 支持点击按钮操作（左滑/右滑按钮）

**设计特点**：
- 卡片跟随手势旋转和位移
- 左滑显示红色"Pass"标签，右滑显示绿色"Yep"标签
- 卡片堆叠效果（最多显示 3 张）

### 4. 餐厅列表（Restaurants）- 数据管理中心
**功能说明**：
- 展示所有餐厅的完整列表
- 支持按分类、价格区间筛选
- 支持按评分、距离排序
- 单个餐厅操作：查看详情、标记/取消拉黑

**设计特点**：
- 顶部固定筛选器和排序器
- 餐厅卡片展示关键信息（图片、名称、评分、距离、价格、标签）
- 拉黑餐厅显示灰色半透明遮罩

### 5. 个人中心（Mine）- 用户管理页
**功能说明**：
- 用户信息展示
- 数据统计（收录餐厅、我添加的、当前可选）
- 添加新餐厅表单（UGC录入）
- 我添加的餐厅列表
- 设置入口
- 关于页面
- 使用贴士

**设计特点**：
- 头部橙色区块与白色统计卡片的层叠效果（负边距 -mt-6，圆角 rounded-3xl）
- 添加餐厅表单支持封面图选择、多标签选择、评分系统
- 设置和关于页面从右侧滑入（motion 侧滑动画）

## 交互说明

### 导航交互
- **点击底部 Tab 栏** → 切换页面（首页、大转盘、卡片滑选、餐厅列表、我的）
- **Tab 栏图标** → 当前页面图标高亮显示（橘子橙 #F97316）

### 首页交互
- **点击"随机替我选一个！"** → 跳转到大转盘页面
- **点击"大转盘"按钮** → 跳转到大转盘页面
- **点击"摇一摇"按钮** → 触发摇晃动画，弹出结果浮层
- **点击"卡片滑"按钮** → 跳转到卡片滑选页面
- **点击热门推荐餐厅** → 暂无详情页，未来可扩展
- **点击"添加新餐厅"** → 跳转到个人中心页面

### 大转盘交互
- **点击"开始抽取"** → 转盘开始旋转，按钮变为"正在转..."
- **转盘停止** → 显示结果卡片（从底部滑入）
- **点击"再转一次"** → 重新旋转转盘
- **点击"就这家了"** → 关闭结果卡片

### 卡片滑选交互
- **手指左滑卡片** → 卡片向左飞出，记录为"不想吃"
- **手指右滑卡片** → 卡片向右飞出，记录为"想吃"
- **点击"✗"按钮** → 等同于左滑
- **点击"♥"按钮** → 等同于右滑
- **滑完所有卡片** → 显示结果页面（随机选择一个"想吃"的餐厅）
- **点击"再玩一次"** → 重置卡片堆栈

### 餐厅列表交互
- **点击分类标签** → 筛选对应分类的餐厅
- **点击价格区间** → 筛选对应价格的餐厅
- **点击排序选项** → 按选定规则重新排序
- **点击餐厅卡片** → 暂无详情页，未来可扩展
- **点击"拉黑"按钮** → 将餐厅标记为拉黑状态
- **点击"取消拉黑"按钮** → 恢复餐厅到可选列表

### 个人中心交互
- **点击"添加新餐厅"** → 从右侧滑入添加表单页面
- **添加表单 - 选择封面图** → 多张预设图片选择
- **添加表单 - 填写信息** → 表单验证，必填项标红
- **添加表单 - 点击"发布"** → 验证通过后添加到餐厅列表，显示成功提示
- **点击"设置"** → 从右侧滑入设置页面
- **点击"关于"** → 从右侧滑入关于页面
- **点击设置页面"✗"** → 返回个人中心
- **点击关于页面"✗"** → 返回个人中心

### 通用交互
- **按钮点击** → scale 缩放反馈（whileTap: scale: 0.97）
- **页面切换** → 无过渡动画（即时切换）
- **滑动列表** → 隐藏滚动条，流畅滚动
- **成功提示** → 从顶部滑入，3 秒后自动消失

## 响应式设计

- **目标设备**: 移动端优先（320px - 768px）
- **布局策略**: 单列布局，全宽容器
- **触控优化**: 按钮最小点击区域 44x44px
- **字体缩放**: 使用相对单位（rem/em），支持系统字体缩放

---

## 动画规范

### 页面转场
- **Tab 切换**: 无动画（即时切换）
- **侧滑页面**: `spring` 弹性动画（stiffness: 300, damping: 30）
- **浮层弹出**: `scale` + `opacity` 渐显动画

### 微交互
- **按钮点击**: `scale(0.97)` 缩小反馈
- **摇一摇**: `rotate` 旋转 + `scale` 缩放组合动画
- **转盘旋转**: `ease-out-quint` 缓动函数
- **卡片滑动**: 跟随手势 `x/y` 位移 + `rotate` 旋转

### 反馈动画
- **成功提示**: 从顶部滑入（y: -60 → 0）
- **结果卡片**: 从底部滑入（y: 100% → 0）

## 数据结构

### Restaurant（餐厅数据模型）
```typescript
{
  id: string;              // 唯一标识
  name: string;            // 餐厅名称
  category: string;        // 分类（川菜、日料等）
  image: string;           // 封面图 URL
  rating: number;          // 评分（1-5）
  distance: string;        // 距离描述
  priceLevel: 1 | 2 | 3;  // 价格档次
  tags: string[];          // 标签数组
  description: string;     // 描述文本
  address: string;         // 地址
  isBlacklisted: boolean;  // 是否拉黑
  isUserAdded?: boolean;   // 是否用户添加
}
```

## 技术栈

- **框架**: React 18 + TypeScript
- **路由**: React Router 7 (Data Mode)
- **动画**: Motion (Framer Motion)
- **样式**: Tailwind CSS v4
- **图标**: Lucide React
- **状态管理**: React Context API
- **Canvas**: 原生 Canvas API（转盘绘制）

## 后续优化方向
