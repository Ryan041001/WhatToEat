## 遇到的问题和解决
1. 问题：项目初期需要同时兼顾后端架构设计、数据库设计和环境初始化，容易出现文档与代码结构脱节。
   解决：先明确“高德 POI 作为外部餐厅主数据、本地数据库只保存用户侧数据”的边界，再据此拆分后端模块和数据库表结构，保证文档与工程目录保持一致。

2. 问题：用户侧数据既要支持黑名单、备注，也要为后续推荐去重预留扩展空间，表结构设计需要兼顾当前作业要求与后续实现。
   解决：设计 `users`、`user_blacklist`、`user_restaurant_note`、`user_choice_history` 四张核心表，并补充唯一约束、外键和索引，同时使用 Flyway migration 管理建表脚本。

3. 问题：后端开发环境除了 Spring Boot 初始化外，还需要提前考虑本地开发、测试环境和数据库迁移的一致性。
   解决：完成 Maven Wrapper、Spring Boot、JPA、Flyway、MySQL/H2 的基础配置，区分 `dev` 和 `test` 配置文件，为后续联调和测试打基础。

4. 问题：默认配置使用 `dev` profile，测试和本地演示在未启动 MySQL 时会直接失败，影响作业验收。
   解决：补充 `test` profile 的 H2 配置，并在测试类中固定使用 `@ActiveProfiles("test")`，保证 `./mvnw test` 可直接通过，同时提供无 MySQL 的后端启动命令用于课堂演示。

## 心得体会
这次软件架构设计让我更清楚地认识到，后端开发不只是写接口，还要先把系统边界、模块职责、数据模型和环境约束定义清楚。先把架构和数据库设计做扎实，后面的接口开发、联调和测试会顺很多。通过这次作业，我也更熟悉了用文档、ER 图和 migration 脚本把设计落到工程结构中的过程。
